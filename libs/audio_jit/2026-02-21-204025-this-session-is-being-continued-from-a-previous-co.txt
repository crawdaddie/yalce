● All changes are in place. Here's a summary of what was done:

  audio_jit.cpp changes:

  1. Two new runtime helpers (after ylc_attach_node_inlet):
    - ylc_get_inlet_node(inputs_raw, inlet_idx) → void* — returns
  inputs[inlet_idx] as Node*, hoisted before the sample loop
    - ylc_bufread(node_raw, phase) → double — lerp read from node->output.buf
  using node->output.size
  2. BufRefHandler (new): takes the symbol name (amen), registers it as a hidden
   inlet in ctx.buf_ref_inputs (so CompileAudioFnHandler wires it via
  ylc_attach_node_inlet), and hoists a call to ylc_get_inlet_node(inputs_ptr, 
  inlet_idx) before the sample loop, returning the !llvm.ptr
  3. BufReadHandler (rewritten): takes (node_ptr, phase) — calls
  ylc_bufread(node_ptr, phase) directly (no more BufReadOp)
  4. Removed the ugly AST_LET tuple bufdata special case
  5. DSP_BUILTIN registrations updated: bufref added, bufread type changed from
  (i64, ptr, f64) → f64 to (ptr, f64) → f64

  The expected usage now works end-to-end:
  let _ = compile_audio_fn (fn () ->
    phasor 0.7 |> bufread (bufref amen)
  ) |> play_node
  - bufref amen → registers hidden inlet, hoists ylc_get_inlet_node call,
  returns Node*
  - bufread node_ptr phase → calls ylc_bufread(node_ptr, phase) for lerp
  playback
